# 使用说明（面向业务/产品同学）

## 本版本新增能力

- Rerank 重排模型：提升最终答案的上下文相关性。
- BM25 + 向量混合检索：在语义匹配外补充关键词匹配，降低漏召回。
- 自动去重：减少重复片段，降低 token 消耗，提升响应速度。
- Web UI 聊天页面：非技术同学可直接网页使用。
- 向量数据库切换为 Chroma：本地免费、无需云账号。
- Docker 沙箱运行：方便跨机器移植。

## 一次完整流程

1. 书籍放入 `data/books`
2. 运行 `python src/ingest.py`（增量入库）
3. 运行 `streamlit run src/web_ui.py` 打开网页问答

## 查询链路

1. 向量召回（Chroma）
2. BM25 召回（本地缓存索引）
3. 分数融合
4. 文本去重
5. Rerank 重排
6. DeepSeek 生成

## 环境变量

- `DEEPSEEK_API_KEY`：DeepSeek API 密钥
- `VECTOR_DB_PATH`：Chroma 本地持久化目录
- `VECTOR_COLLECTION`：向量集合名
- `EMBED_MODEL`：Embedding 模型
- `RERANK_MODEL`：重排模型
- `VECTOR_RECALL_K`：向量候选召回数
- `BM25_RECALL_K`：BM25 候选召回数
- `TOP_K`：进入生成阶段的最终上下文数量

## Docker 使用

- `docker compose up --build -d`：启动沙箱
- `docker compose exec app python src/ingest.py`：执行增量入库
- 浏览器访问 `http://localhost:8501`

## 速度建议

- 减小 `VECTOR_RECALL_K` 和 `BM25_RECALL_K`
- 减小 `TOP_K`
- 使用 GPU 运行 rerank（可选）
